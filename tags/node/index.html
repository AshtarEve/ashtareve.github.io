关于 node.js 的版本

本文参考 node.js 官网发布页面，并添加一些解释。

发布

主版本的 Node.js 进入 当前版(Current) 将持续六个月的时间，在此期间库作者可以对其进行支持。 六个月之后，奇数版本（诸如 9、11 等）将变为不支持状态，只有偶数版本（诸如 10、12 等）变成 活跃 LTS (Active LTS) 状态，并且准备投入使用。 LTS 发布版的状态是“长期支持版(long-term support)”，这意味着重大的 Bug 将在后续的 30 个月内持续得到不断地修复。 上线应该仅使用 活跃 LTS (Active LTS) 或者是 维护 LTS (Maintenance LTS) 版。

通常，node.js 每年 4 月份发布偶数版本，10 月份发布奇数版本。

概念

  1. Current: 当前最新的版本，该阶段通常持续六个月的时间，库作者可以对其进行支持。
  2. Active LTS：指正在积极维护和升级的版本系列，包括向后移植非破坏性功能和改进，解决错误以及修补安全漏洞。
  3. Maintenance LTS：指维护的 LTS 版本，更加稳定，直到它的生命周期终止，只会在短时间内收到错误修复和安全补丁。
  4. End-of-life：进入到 EOL 时间线的版本，将不在维护。
