使用 denyhosts 防止 ssh 暴力破解

使用 denyhosts 防止 ssh 暴力破解

发现问题

使用 ssh 远程登陆，发现有失败登陆记录了上百次：

使用命令查看近期登陆失败的记录

解决方法

将 ip 加入 /etc/hosts.deny

ALL: xxx.xxx.xxx.xxx

服务器会直接 refused 掉该 ip 的所有请求
(cat /var/log/secure)

使用工具 denyhosts

下载安装 denyhosts

配置 denyhosts

SECURE_LOG = /var/log/secure
BLOCK_SERVICE = ALL
ADMIN_EMAIL =

配置需要注意 lock 文件配置需要和 daemon-control 的 DENYHOSTS_LOCK 文件一致，不修改这个配置即可保持一致。

启动&设置开机启动 denyhosts

查看启动状态&

denyhosts 问题

denyhosts 只对密码输入错误的进行检查，添加进了 hosts.deny 文件
对于禁用密码登陆，使用ssh密钥免密登陆的没有添加进 hosts.deny 文件

解决方法，使用 bash 脚本直接将 lastb 中登陆失败的 ip 加入 /etc/hosts.deny

创建 bash 脚本：vim /root/cron/denyhosts.sh

将脚本加入系统定时执行 cron
